# Imagify - AI Image Generator

Imagify is a full-stack web application that allows users to generate high-quality images from text descriptions using powerful AI models. Built with the MERN stack (MongoDB, Express.js, React, Node.js), it integrates Clipdrop API for image generation and Razorpay for seamless credit purchases.

## Project Overview & Workflow

The core functionality of Imagify revolves around a credit-based system. Users register/login, receive initial free credits, and can use these credits to generate images. When a user runs out of credits, they can purchase more via a Razorpay payment gateway integration.

### High-Level Workflow
1. **Authentication:** Users sign up or log in. A JWT (JSON Web Token) is generated by the backend and stored in the frontend (usually in context/local storage) to authenticate subsequent requests.
2. **Credit System:** Each image generation costs a certain number of credits. The backend verifies the user's credit balance before proceeding with the API call.
3. **Image Generation:** The user enters a text prompt. The frontend sends this prompt to the backend. The backend then makes a secure API request to the **Clipdrop API**, receives the generated image (usually as an arraybuffer or base64 string), and sends it back to the frontend to be displayed.
4. **Payment Integration:** If credits are low, the user navigates to the "Buy Credits" page. Selecting a plan triggers a Razorpay order creation from the backend. The frontend handles the Razorpay checkout overlay. Upon successful payment, Razorpay sends a success signature, which the backend verifies to update the user's credit balance.

## Architecture & Tech Stack

**Frontend:**
- **React.js (via Vite):** For building a fast, component-based user interface.
- **Tailwind CSS:** For highly responsive and modern styling.
- **Framer Motion (`motion`):** For adding smooth animations and page transitions.
- **React Router Dom:** For handling client-side routing (`/`, `/buy`, `/result`).
- **Context API:** Used for global state management (e.g., user authentication state, current credit balance).
- **Axios:** For making HTTP requests to the backend API.

**Backend:**
- **Node.js & Express.js:** For building the RESTful API and handling server logic.
- **MongoDB & Mongoose:** For storing user data, transaction history, and credit balances.
- **JWT & Bcrypt:** For secure password hashing and user authentication.
- **Clipdrop API:** The core AI engine used to convert text prompts into images.
- **Razorpay:** For processing secure payments in INR.

## How It Works (Step-by-Step)

### 1. Generating an Image
- User types a prompt in the `<Result />` or `<Home />` component.
- The `generateImage` function is called via Context API/Axios.
- The backend route (`/api/image/generate`) receives the request.
- Middleware verifies the JWT token to identify the user.
- The controller checks if the user has enough credits in MongoDB.
- Let's say yes—the server makes a `fetch/axios` call to the Clipdrop API passing the prompt.
- Clipdrop returns the image buffer.
- The backend converts this buffer to a base64 string and sends it to the frontend.
- User's credit balance is decremented in the database by 1.
- The base64 string is rendered in an `<img>` tag on the frontend.

### 2. Buying Credits
- User navigates to the `/buy` route.
- User clicks "Purchase" on a specific plan.
- The backend (`/api/user/pay`) initializes a Razorpay order using `razorpay.orders.create()`.
- The frontend opens the Razorpay checkout modal with the returned `order_id`.
- User completes the payment.
- The frontend receives `razorpay_payment_id`, `razorpay_order_id`, and `razorpay_signature` and sends them to `/api/user/verifyRazorpay`.
- The backend uses the `crypto` module to hash the payment ID and order ID with the Razorpay Secret to verify the signature.
- If verified, the user's credit balance is updated in MongoDB and a transaction record is created.

## Features
1. **Text-to-Image Generation:** Convert textual ideas into vivid images.
2. **Credit-Based System:** Secure management of usage limits per user.
3. **Payment Gateway Integration:** Real-world payment processing using Razorpay.
4. **Authentication:** Secure JWT-based user login and registration.
5. **Responsive Design:** Beautifully styled with Tailwind CSS to work on any device.

## Environment Variables

To run this project, you will need to add the following environment variables to your respective `.env` files.

**In frontend (`frontend/.env`):**
```env
VITE_BACKEND_URL='http://localhost:5000'
VITE_RAZORPAY_KEY_ID="your_razorpay_key_id"
```

**In Backend (`backend/.env`):**
```env
MONGO_URI="your_mongodb_atlas_uri"
JWT_SECRET="your_jwt_secret_key"
CLIPDROP_API="your_clipdrop_api_key"
RAZORPAY_KEY_ID="your_razorpay_key_id"
RAZORPAY_KEY_SECRET="your_razorpay_secret_key"
CURRENCY="INR"
```
